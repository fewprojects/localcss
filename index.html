Introduction

Suppose you're working with any backend language/Javascript library or any other programming language on your localhost that requires Web front-end(HTML, CSS) to work with. For example : ReactJS, VueJS, AngularJS, MERN stack, MEAN stack, PHP etc.

You usually build up backend first on your localhost and to work on it with default html without styling gives you extra work to add styles to the document.

To get rid of default styling, you may want to use bootstrap like css frameworks, but the problem with bootstrap and other Similar CSS frameworks is that you need to write class-atributes to style the element and that adds you an extra work to do.

Solution of this problem is to use Sementic CSS styling frameworks

With local.css you don't need to style add any element of HTML to ease your work with localhost

What is sementic styling?

Containers

Containers are required to seprate website components.

You have to use class .container for div element to make it a container.

Containers in local.css are given padding of 1.5rem which is essential to give white space between components

There are three kinds of containers are available here : default container, container-1280, container-980

Default Container

It is flexible container with 100% width. It takes full-width of the parent element

Default Container 
<div class="container">Default Container</div>

Container 1280

Container 1280 is a continer with max-width of 1280px, that means if screen-size exceeds 1280px, it will maintain fixed width of 1280px.

If screen size is less than 1280px, it will aquire full width of the screen and work like a default container

Container 1280 
<div class="container-1280">Container 1280</div>

Container 980

Container 980 is a continer with max-width of 980px, that means if screen-size exceeds 980px, it will maintain fixed width of 980px.

If screen size is less than 980px, it will aquire full width of the screen and work like a default container

Container 980 
<div class="container-1280">Container 980</div>

Try minimizing browser's screen using CTRL - up to 50% to see containers in action.
